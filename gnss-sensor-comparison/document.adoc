== 0. 목적 및 개요 (Purpose & Scope)

* 본 계획서는 NEUBOAT에 적용하기 위해 GNSS 항법 센서 4종 — Raymarine AR200(기존 사용 모델), Garmin MSC-10(듀얼 안테나), Garmin GPS 24xd(Valhalla 설치 모델), Hemisphere Vector V200n(Hinckley 설치 모델) — 의 출력 품질을, 정확도 기준(reference) 센서인 VectorNav VN-300(기존 아비커스 2호 설치 모델)을 레퍼런스로 하여 동일 조건에서 비교·검증하기 위한 계획을 정의한다.
* 노이즈(A)·속도 일관성(B)·정지 산포\(C)의 3개 평가 항목과 NMEA 2000 PGN 수신 rate 안정성은 정답(ground truth)이 없는 reference-free 자기일관성 방식으로 비교하고, 위치·heading·속도의 정확도(D)는 고정밀 GNSS/INS인 VectorNav VN-300을 기준(reference)으로 평가한다.
* 평가 결과는 단일 종합 점수로 환원하지 않고, 시나리오(정지 / 주행 / 도킹 / 선회) × 평가 기준(A~D)의 매트릭스로 판정한다. 최종적으로 AR200과 유사하거나 더 나은 성능을 보이는 센서를 선정하여 NEUBOAT dock control·navi control 적용까지 확장한다.

=== 설치 및 고정

* 안테나 4종을 내부 공간이 있고 천공이 용이한 FIBOX 인클로저에 장착한다.
* 인클로저를 아비커스 2호 천장 프로파일에 고정하여 4종 안테나의 상대 위치를 일정하게 유지한다.
* 정확도 기준인 VN-300은 기존 아비커스 2호 설치 위치를 그대로 사용한다.

=== 로깅 방식 (테스트 2회)

* 1회차: 로컬 PC에서 Kvaser로 NMEA 2000(CAN)과 VN-300을 동시 로깅하며, 시간 동기화(time sync)는 GPS time 기준으로 맞춘다.

NOTE: NMEA 2000으로 수신되는 GPS time의 단위는 10⁻⁴ s(0.1 ms)인 반면 VN-300은 ns 단위이므로, 해상도 차이를 감안해 최대한 정렬(best-effort)하여 동기화한다.

* 2회차: ORU에서 로깅한다. (SP·FGG 모두 로깅 수행) 1회차와 동일하게 VN-300을 함께 로깅할 수 있도록 준비하여 진행한다.

== 3. 평가 항목

_노이즈(A)·속도 일관성(B)·정지 산포\(C)의 3개 평가 항목은 정답(ground truth)이 없는 reference-free 자기일관성 방식으로 평가하고, 위치·heading·속도의 정확도(D)는 고정밀 GNSS/INS인 VN-300을 기준(reference)으로 평가한다._

상대 비교의 기준선(baseline)은 기존 사용 모델인 AR200으로 두며, A·C 지표는 AR200 대비 개선율(%)로 표기한다. 절대 정확도의 기준(reference)으로는 고정밀 GNSS/INS인 VectorNav VN-300을 사용하여, 위치·heading·속도의 정확도(D 그룹)를 VN-300 출력 대비 RMSE로 평가한다. B2~B5은 2회차 ORU를 사용하여 로깅할 때만 평가한다.

=== 노이즈 (jerk) 산출

* A1·A2 jerk는 위치·속도 시계열의 2차 차분으로 산출한다 — 2차 차분이 등속 운동 성분을 자동 상쇄하므로, 별도 추세(trend) 추정 없이 고주파 잔차(노이즈)만 남는다.
* jerk·노이즈는 등속 직진(일반 주행, 10노트 내외) 구간에서 산출하고, 선회·가감속 구간은 제외한다 — 실제 기동의 가속도 변화가 노이즈로 오인되지 않도록 한다.

=== 버스 부하·ISO 요청 응답 측정

* 정상 부하뿐 아니라 버스 부하(busload) 50%·70% 조건에서도 수신 rate를 측정한다 — Kvaser의 트래픽 생성(traffic generation) 기능으로 버스 부하를 인가한다.

== 4. 종합 판정 및 한계

* 판정은 단일 종합 점수로 환원하지 않고, 시나리오(정지/주행/도킹/선회) × 평가 기준(A/B/C/D) + PGN rate σ 로 본다.
* A~C는 상대·자기일관성 지표이며, 위치·heading·속도의 정확도(D)는 VN-300을 기준(reference)으로 평가한다.
* 지연(lag)·jitter는 PGN 수신 Rate 평가(σ·jitter)에서 다룬다.

== 5. 향후 계획 — 선정 센서 적용

*Dock Control · Navi Control 적용*

본 검증 결과를 바탕으로 선정한 센서를 NEUBOAT에 적용하여 dock control과 navi control을 수행한다.

=== 센서 선정 기준

* 기존 사용 모델인 AR200과 유사하거나 더 나은 성능을 보이는 센서를 우선 선정한다.
* A~C 자기일관성 지표와 D(VN-300 대비 위치·heading·속도 정확도)에서 AR200 이상으로 확인된 센서를 후보로 한다.
